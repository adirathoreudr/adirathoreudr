### 🤖 **AIOps for Kubernetes** — Autonomous Incident Commander
**Live:** [GitHub](https://github.com/adirathoreudr/aiops-kubernetes)

AI agent that handles K8s incidents **without waking you up**. Detects anomalies, diagnoses root cause, executes fixes, and logs everything.

```
Python | LangChain | OpenAI | Kubernetes | ArgoCD | Prometheus | Loki
```

**The Problem It Solves:**
- 🚨 Alert fatigue: engineers ignore 70% of alerts (real stat)
- ⏱️ MTTR: takes 45 min to diagnose and fix an issue
- 👤 On-call rotation: exhausting, error-prone

**The Solution:**
1. Prometheus/Grafana detect anomaly → fires alert
2. LLM reads logs, metrics, context, alert history
3. AI reasons: "OOM killed pod. Scale deployment up + increase memory request"
4. Execute via ArgoCD: deployment patched, pod restarted
5. Human reviews decision in morning, with full audit trail

**Results:**
- 📉 **Alert noise:** 50% reduction (smart deduplication)
- ⚡ **MTTR:** 60% faster (auto-resolve 70% of critical incidents)
- 😴 **On-call:** fewer false alarms, smarter routing
